APPLIED AI LEARNING SUMMARY – RAG, LANGCHAIN, AND SYSTEM DESIGN

  -----------------------------
  1. OVERALL LEARNING JOURNEY
  -----------------------------

You transitioned from understanding basic AI concepts to building a full
Retrieval-Augmented Generation (RAG) system.

Key shift: - From “using LLMs” → “building AI systems”

You now understand: - Components (loaders, embeddings, vector DBs,
LLMs) - Flow (data → retrieval → reasoning → answer) - Tradeoffs (OpenAI
vs Ollama vs Hugging Face) - System thinking (FDSE mindset)

  --------------------------------
  2. CORE RAG PIPELINE (PHASE 1)
  --------------------------------

Flow:

Data Source (Website / PDF) ↓ Loader (WebBaseLoader) ↓ Documents ↓ Text
Splitter (RecursiveCharacterTextSplitter) ↓ Chunks ↓ Embeddings (OpenAI
/ HuggingFace / Ollama) ↓ Vector Database (FAISS / Chroma) ↓ Retriever /
Similarity Search ↓ Relevant Chunks

Key Insights: - Splitting improves retrieval accuracy - Embeddings
convert meaning into vectors - Vector DB enables fast similarity
search - Retriever abstracts database access

  --------------------------
  3. LIMITATION OF PHASE 1
  --------------------------

You built: → Semantic Search System

But NOT: → AI Answering System

Problem: - Output = chunks, not answers - No reasoning layer

  ------------------------------
  4. FULL RAG SYSTEM (PHASE 2)
  ------------------------------

Flow:

User Question ↓ Retriever ↓ Relevant Chunks ↓ StuffDocumentsChain ↓
Prompt Template (with {context}) ↓ LLM (ChatOpenAI / Groq / etc.) ↓
Output Parser ↓ Final Answer

Key Insight: Retriever = “find information” LLM = “reason over
information”

  -------------------------
  5. IMPORTANT COMPONENTS
  -------------------------

Document Chain: - Takes input + context - Builds prompt - Calls LLM

Retriever: - Fetches relevant chunks - Independent of vector DB
implementation

Retrieval Chain: - Combines retriever + document chain - Automates full
RAG flow

  -----------------------------
  6. PROMPT ENGINEERING SHIFT
  -----------------------------

Two modes:

1.  LLM-only: “You are an expert…”

-   Uses training knowledge
-   Risk of hallucination

2.  RAG-based: “Answer using only provided context”

-   Uses retrieved data
-   More reliable

Key Concept: LLM-only → Think RAG → Look up + Think

  ------------------------
  7. EMBEDDING PROVIDERS
  ------------------------

OpenAI: - High quality - Paid - Hosted

Ollama: - Local runtime - Private - Offline capable

Hugging Face: - Model ecosystem - Provides many models - Supports
embeddings + LLMs

Important: Hugging Face = platform, not a single model

  ---------------------
  8. VECTOR DATABASES
  ---------------------

FAISS / Chroma:

Responsibilities: - Store vectors - Index vectors - Perform similarity
search

Key Insight: Not just storage → Search engine for vectors

  -------------------
  9. LANGCHAIN ROLE
  -------------------

LangChain = orchestration framework

Provides: - Loaders - Splitters - Prompt templates - Chains -
Integrations

Does NOT: - Create embeddings - Store vectors

  ------------------------------------------
  10. LCEL (LANGCHAIN EXPRESSION LANGUAGE)
  ------------------------------------------

Concept: Pipeline composition

Example:

input → prompt → model → parser input → retriever → prompt → model →
parser

Purpose: - Build pipelines cleanly - Standardize chaining logic

  ----------------------
  11. OOP IN LANGCHAIN
  ----------------------

Used concepts:

Abstraction: - Hides API complexity

Polymorphism: - Swap models (OpenAI, Ollama, Groq)

Encapsulation: - Internal logic hidden

Classes used: - ChatPromptTemplate - ChatOpenAI - StrOutputParser -
Retriever - Chains

  --------------------------
  12. COMPLETE SYSTEM VIEW
  --------------------------

Final Architecture:

User Query ↓ Retriever ↓ Relevant Chunks ↓ Context Injection ({context})
↓ Prompt Template ↓ LLM ↓ Parsed Output ↓ Answer

  -----------------------------------
  13. KEY INSIGHTS (MOST IMPORTANT)
  -----------------------------------

1.  Retrieval quality drives system quality
2.  LLM is NOT the source of truth in RAG
3.  Context = critical input to LLM
4.  LangChain is optional (not mandatory)
5.  System design > coding syntax

  ------------------------
  14. FINAL MENTAL MODEL
  ------------------------

load → split → embed → store → retrieve → prompt → llm → answer
